About This Project

A personal experiment. How it was built and why.

Motivation

Advanced Squad Leader has one of the most complex rulebooks of any board game — over 1 million tokens, densely cross-referenced, with numerous tables, exceptions, and numerical calculations. Looking up rules mid-game is slow and error-prone even for experienced players.

This is a personal project I built for my own use while learning the game. It is not a polished product — answers can be wrong, citations can be off, and the system struggles with questions that require chaining multiple lookups.

The core question is cost-effectiveness: a large frontier model like gpt-5.4 can answer most of these questions accurately, but costs around $0.06 per query. Mini-class models cost under $0.01. The experiment is whether a cheaper model can get close enough to be practically useful.

System Architecture

FastAPI · OpenAI Responses API · WebSocket streaming · PostgreSQL · AI Judge evaluation

The backend is a FastAPI application running on Digital Ocean behind nginx. Chat is delivered over a persistent WebSocket connection, streaming tokens to the browser as they are generated.

Each question flows through a four-step pipeline:

  • Retrieve & Generate — The question is sent to the OpenAI Responses API with a file_search tool over a vector store of the rulebook. Up to 20 chunks are retrieved and injected into a structured system prompt that instructs the model to cite specific rule section numbers (e.g. A4.34). The response streams token-by-token over WebSocket; the frontend renders markdown progressively and tracks TTFT, RAG latency, total response time, token counts, and estimated cost per query. Conversation history is included for multi-turn context.
  • Validate (optional) — Rule citations in responses are rendered as clickable links that open an in-browser PDF viewer at the exact rulebook page, using a pre-built section-to-page index, so the user can check the answer against the source.
  • Evaluate — Structured eval sets are run against the model and scored by a zero-shot AI Judge as Pass, Fail, or Needs Review. Flagged responses are reviewed manually to confirm or correct the judge's verdict, giving a human-verified accuracy signal across model versions. Results are broken down by question type (Recall vs. Calc).
  • Curate — Every chat interaction is logged. Promising user Q&A pairs are triaged through a review UI into eval sets or SFT training data, feeding the loop that measures and improves the model over time.

Why This Domain Is Challenging

Several properties of the ASL rulebook make it harder than typical RAG use cases:

  • Scale — At 1M+ tokens, the rulebook is ~2.5× the length of Deep Learning by Goodfellow et al. Standard context windows cannot hold it.
  • Cross-references — Rules routinely reference other rules by section number. A complete answer often requires retrieving from multiple non-adjacent sections.
  • Multimodal content — The original document contains hex maps, counter diagrams, and fire tables. Converting to text loses visual context that some rules depend on.
  • Precision — Answers that are mostly right but miss an exception or modifier produce incorrect gameplay. Vague answers have no value here.
  • Calculations — Many questions require chaining lookups across multiple tables (terrain effects, unit stats, fire columns) that pure retrieval cannot resolve alone.

Evaluation & Curation Loop

Model performance is tracked using a structured eval pipeline. Each eval consists of a question and a reference answer. After running a model against the eval set, a zero-shot AI Judge scores each response as Pass, Fail, or Needs Review. Results flagged for review are assessed manually.

Evals are split by question type — Recall (pure rule lookup) and Calc (recall + computation) — since these require different capabilities and tend to show different failure modes.

Real production questions feed back into this loop. A review UI surfaces logged user questions alongside the production answer; promising ones are imported into eval files or SFT training data, while duplicates and rejects are hashed and filtered so they never reappear. This keeps the eval set grounded in what users actually ask.

About Advanced Squad Leader

Advanced Squad Leader is a tactical-level hex-and-counter wargame covering WW2 and the Korean War, published by Avalon Hill in 1985. It is widely regarded as the most comprehensive and detailed squad-level wargame ever published. See the Wikipedia entry.
