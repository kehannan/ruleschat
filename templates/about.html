About the project

How it was built,
and why.

A personal experiment. Not a polished product. Answers can be wrong, citations can be off, and the system struggles with questions that chain multiple lookups. The goal is to find out whether a cheaper model can get close enough to a frontier model to be practically useful — and to publish what it gets wrong as evidence either way.

01
Motivation

A 1M-token rulebook and
a cost question.

Advanced Squad Leader has one of the most complex rulebooks of any board game — over 1 million tokens, densely cross-referenced, with numerous tables, exceptions, and numerical calculations. Looking up rules mid-game is slow and error-prone even for experienced players.

A large frontier model like gpt-5.4 answers most of these questions accurately, but costs around $0.06 per query. Mini-class models cost under $0.01. The experiment is whether a cheaper model can get close enough to be useful.

This is a personal project I built for my own use while learning the game. It is not a polished product — answers can be wrong, citations can be off, and the system struggles with chained lookups.

02
Architecture

FastAPI · OpenAI Responses
· WebSocket · Postgres.

Backend runs on Digital Ocean behind nginx. Chat streams over a persistent WebSocket. Each query flows through a four-step pipeline; the same one described on the home page.

  1. i
    Retrieve & Generate
    Question is sent to the OpenAI Responses API with a file_search tool over a vector store of the rulebook. Up to 20 chunks injected into a structured system prompt that instructs the model to cite section numbers. Response streams over WebSocket; client tracks TTFT, RAG latency, total time, token counts, and per-query cost.
  2. ii
    Validate (optional)
    Citations in responses are rendered as clickable links that open an in-browser PDF viewer at the exact rulebook page, using a pre-built section-to-page index, so the user can check the answer against the source.
  3. iii
    Evaluate
    Structured eval sets are run against the model and scored by a zero-shot AI Judge as Pass / Fail / Needs Review. Flagged responses are reviewed manually to confirm or correct the judge's verdict. Results broken down by question type (Recall vs. Calc).
  4. iv
    Curate
    Every chat interaction is logged. Promising user Q&A pairs are triaged through a review UI into eval sets or SFT training data, feeding the loop that measures and improves the model over time.
03
What makes this hard

Where this domain bites the
standard RAG playbook.

Scale At 1M+ tokens, the rulebook is ~2.5× the length of Deep Learning by Goodfellow et al. Standard context windows can't hold it.
Cross-references Rules routinely reference other rules by section number. A complete answer often requires retrieving from multiple non-adjacent sections.
Multimodal content The document contains hex maps, counter diagrams, and fire tables. Text conversion loses visual context that some rules depend on.
Precision Mostly-right answers that miss an exception or modifier produce incorrect gameplay. Vague answers have no value here.
Calculations Many questions require chaining lookups across multiple tables (terrain, unit stats, fire columns) that pure retrieval can't resolve alone.

About ASL

A 1985 hex-and-counter
tactical wargame.

Advanced Squad Leader is a tactical-level hex-and-counter wargame covering WW2 and the Korean War, published by Avalon Hill in 1985. It is widely regarded as the most comprehensive and detailed squad-level wargame ever published. See the Wikipedia entry.
